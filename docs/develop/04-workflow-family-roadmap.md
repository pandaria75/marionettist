# Workflow Family Roadmap

Marionettist's long-term direction is Agent Workflow Harness, not only coding harness. Workflow families should evolve through real usage, not speculative platform work.

```text
Marionettist
├── coding workflows
│   ├── feature development
│   ├── bugfix
│   ├── code review
│   └── refactor
│
├── writing workflows
│   ├── technical blog
│   ├── project promotion
│   ├── research article
│   └── script writing
│
├── research workflows
│   ├── web research
│   ├── source comparison
│   └── evidence summary
│
└── ops workflows
    ├── server diagnosis
    ├── docker compose repair
    └── deployment check
```

## Coding workflows

- Goal: make repository changes safer through planning, slicing, review, repair, validation, and gates.
- Typical scenarios: feature development, bugfix, refactor, review, validation follow-up.
- Possible skills: implementation planning, bounded coding, diff review, validator routing, knowledge update.
- Possible artifacts: requirements, implementation plan, context pack, changed-file inventory, validation evidence.
- Possible checkpoints: analysis gate, slice gate, pre-done critic gate, final approval.
- Current-stage development: yes; this remains the primary implemented family.

## Writing workflows

- Goal: produce high-quality prose through structured brief, source, outline, draft, review, repair, and final steps.
- Typical scenarios: technical blog, project promotion article, research article, script writing.
- Possible skills: brief builder, source collector, outline planner, draft writer, style reviewer, repair editor.
- Possible artifacts: brief, sources, outline, draft, review notes, final Markdown, style profile.
- Possible checkpoints: brief approval, outline approval, source sufficiency, draft review, final editorial pass.
- Current-stage development: not in Marionettist mainline; explored through Quill dogfooding.

## Research workflows

- Goal: gather, compare, and summarize evidence with traceable sources.
- Typical scenarios: web research, source comparison, evidence summary, technology evaluation.
- Possible skills: query planning, source triage, evidence extraction, comparison matrix, synthesis review.
- Possible artifacts: source list, evidence table, comparison notes, summary, uncertainty log.
- Possible checkpoints: source quality check, evidence coverage check, contradiction review.
- Current-stage development: future research; do not implement before stronger evidence.

## Ops workflows

- Goal: support safe operational diagnosis and repair without hiding risky actions.
- Typical scenarios: server diagnosis, docker compose repair, deployment check, incident triage.
- Possible skills: log triage, config inspection, risk classification, repair planning, smoke validation.
- Possible artifacts: incident summary, command log, diagnosis notes, repair plan, validation evidence.
- Possible checkpoints: destructive-command approval, mitigation review, post-repair validation.
- Current-stage development: future research; dangerous operations require explicit human control.
