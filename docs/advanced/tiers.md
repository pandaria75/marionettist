# Tiers

Tiers describe how much workflow structure a task needs.

## Tier summary

- **Tier S** — trivial, low-risk work
- **Tier M** — standard scoped work that still needs context
- **Tier L** — complex or boundary-sensitive work that needs requirements, planning, and approved slices

## Why tiers exist

Marionettist should scale process to risk. Small work should stay light. Larger work should gain stronger task artifacts and stronger stop points.
